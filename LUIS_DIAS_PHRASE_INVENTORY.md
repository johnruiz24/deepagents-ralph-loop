## FINAL CHECKLISTS

### Phrases to Use (Proven)
- "This article takes a different approach"
- "From Prototype to Production"
- "This is where most tutorials end — and where the real engineering begins"
- "Your Move"
- "We are bridging the gap from X to Y"
- "The question isn't whether... it's whether..."
- "That's the difference between X and Y"
- "for a user waiting..." (visceral moment)
- "At enterprise scale, these savings compound"
- "For the most common use case..."

### Phrases to Avoid (Not Luis's Style)
- "Just use X, it's easy!" (too casual)
- "Obviously..." (condescending)
- "As you probably know..." (assumes knowledge)
- "In conclusion..." (dated academic marker)
- "Thanks for reading!" (too warm for this tone)

### Signature Structure Markers
- Always 4 items in lists (capabilities, requirements, bullets)
- Code blocks are 8-15 lines maximum
- Transitions happen via section headers, not connective sentences
- Numbers appear in nearly every performance claim
- Enterprise/scale context mentioned multiple times
- Audience voice is "expert speaking to equals"

## USAGE TIPS

### When Starting Ralph Article
1. Copy the opening hook template
2. Fill in [BRACKETS] with Ralph-specific content
3. Keep the structure exactly
4. Test your opening against: "Does this establish sophistication, then problem, then promise?"

### When Writing Technical Sections
1. Use Template 3 pattern: Narrative → Code (8-15 lines) → Explanation (4 bullets)
2. Every major component gets explained this way
3. Never just show code; always contextualize before and after

### When Adding Proof
1. Start with numbers (Template 5, Level 1)
2. Translate to real-world (Level 2)
3. Scale to enterprise (Level 3)
4. Explain the principle (Level 4)
5. Always cite the number in at least 2 places

### When Concluding
1. Reaffirm accomplishment with numbers
2. Position as part of larger trend
3. Provide toolkit inventory
4. End with "Your Move" section
5. Close with unanswered rhetorical question
