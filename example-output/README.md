# Example Output Structure

This directory demonstrates the output structure generated by Ralph when running the newsletter generation pipeline.

## Directory Layout

Each run creates a timestamped directory with the following structure:

```
example-output/
└── 20260330_001325_Universal_Commerce_Protocol/
    ├── input/
    │   ├── user_prompt.json                    # Original user request
    │   └── topics_and_subtopics.json          # Parsed topics structure
    │
    ├── research/
    │   ├── research_plan.json                 # Query formulation output
    │   ├── raw_data/                          # Raw articles from sources
    │   │   ├── subtopic_1/
    │   │   ├── subtopic_2/
    │   │   └── subtopic_3/
    │   └── tui_strategy_summary.md            # Strategic analysis output
    │
    ├── content/
    │   ├── draft_article.md                   # Initial synthesis
    │   └── final_article.md                   # HBR-edited article
    │
    ├── visuals/
    │   ├── chart_1_market_trends.png         # Dynamically named charts
    │   ├── chart_2_adoption_curve.png
    │   ├── chart_3_competitive_analysis.png
    │   ├── chart_4_timeline.png
    │   ├── chart_5_architecture.png
    │   └── asset_manifest.json               # Chart metadata
    │
    ├── multimedia/
    │   └── narration_professional.mp3        # Audio narration (300+ seconds)
    │
    ├── final_deliverables/
    │   ├── newsletter_final.pdf               # 300 DPI print-ready
    │   ├── newsletter_final.html              # Responsive web version
    │   ├── newsletter.zip                     # Complete package
    │   └── manifest.json                      # Deliverable metadata
    │
    ├── state.json                            # Complete pipeline state snapshot
    └── state_snapshot.json                   # Final state at completion
```

## File Descriptions

### Input Phase
- **user_prompt.json**: Original topic, audience, key concepts
- **topics_and_subtopics.json**: Parsed research structure (3 subtopics × 5 queries)

### Research Phase
- **research_plan.json**: Query formulation agent output (15 queries across sources)
- **raw_data/**: 12 articles (4 per subtopic) fetched from sources
- **tui_strategy_summary.md**: Strategic context analysis (850+ lines)

### Content Phase
- **draft_article.md**: Initial synthesis (3,100+ words)
- **final_article.md**: HBR-edited version (2,200+ words), quality score 70/100

### Production Phase
- **chart_*.png**: 5 professionally formatted charts (300 DPI)
  - Market trends, adoption curves, competitive analysis, timelines, architectures
  - Charts are dynamically generated based on content (not hardcoded!)
- **narration_*.mp3**: Professional audio narration (MP3, 529+ seconds)
- **Newsletter_final.pdf**: Complete publication (PDF, 461+ KB)
- **newsletter_final.html**: Web-responsive version
- **newsletter.zip**: Complete package with all assets

### State Management
- **state.json**: Full pipeline state at each phase
- **state_snapshot.json**: Final state for reproducibility

## Running Ralph to Generate Output

To generate your own output:

```bash
cd /path/to/deepagents-ralph-loop

# Run with default topic
PYTHONPATH=. python scripts/run_e2e_test.py

# Or with custom topic
PYTHONPATH=. python scripts/run_cli.py
```

Output will be created in the `output/` directory (not tracked by git).

## Why Example Output is in Repo

- **Demonstrates structure** for users implementing custom workflows
- **Shows expected outputs** so users know what to expect
- **Provides reference** for state JSON structure
- **Not tracked**: Actual generated outputs go to `output/` (gitignored)

## Reproducibility

State snapshots enable reproducibility:
1. Save `state.json` from a run
2. Pass it to orchestrator to resume from that point
3. Useful for debugging or customization
